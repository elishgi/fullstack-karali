📋 חזון האפליקציה (מתוקן לפי ההסבר שלך):
🏷️ שם: Karali Events Tracker
📱 מסך 1 — רשימת האירועים (דף הבית)
🔹 מה מוצג:

רשימה של לחצנים.

על כל לחצן מופיע:

✅ שם האירוע.

✅ מספר הפעמים שהאירוע קרה.

🔹 מה ניתן לעשות:

לחיצה על לחצן האירוע → מגדילה את המונה (כלומר: מעדכנת את האירוע → +1 occurrence count).

למעלה או למטה → כפתור ➕ להוספת אירוע חדש.

➕ הוספת אירוע חדש:
טופס הוספה:

שם האירוע.

(מונה יתחיל מ־0).

לאחר ההוספה:

מופיעה התראה → "אירוע חדש נוסף".

האירוע יתווסף לרשימה.

📝 עריכת אירוע / מחיקת אירוע:
אפשר להיכנס למסך עריכה:

לשנות את שם האירוע.

לאפס או לשנות את מספר הפעמים שקרה.

לצלם תמונה לאירוע ולהוסיף (פיצ'ר צילום — חובה אצלך).

למחוק את האירוע.

כל פעולה (עריכה / מחיקה / הוספה / הגדלת מונה) → נכנסת ללוגים.

🗺️ שימוש במיקום (פיצ'ר מכשיר 2):
בעת הרחבת אירוע (במסך העריכה), נוסיף אפשרות:
"הוסף מיקום" — לוקח את המיקום הנוכחי ומוסיף אותו לפרטי האירוע.

📄 מסך 2 — היסטוריית פעולות (לוגים)
רשימה ממוחזרת.

כל שורה מציגה:

איזה פעולה בוצעה (הוספה, עריכה, מחיקה, הגדלת מונה).

תאריך פעולה.

פרטי האירוע (שם).

🚀 פיצ'רים מיוחדים:
✅ פיצ'ר 1: צילום תמונה לאירוע → יתווסף במסך עריכה.
✅ פיצ'ר 2: הוספת מיקום → יתווסף במסך עריכה.

📐 מבנה כללי:
דף הבית → רשימת לחצנים (אירועים).

דף הוספה → טופס הוספת אירוע.

דף עריכה → עריכת פרטי אירוע + צילום + מיקום.

דף לוגים → היסטוריית פעולות.

🔄 תקשורת מול השרת:
Events (טבלה קיימת אצלך):
GET /events → לקבל את כל האירועים.

POST /events → להוסיף אירוע.

PUT /events/:id → לעדכן (מונה / שם / תמונה / מיקום).

DELETE /events/:id → למחוק.

Logs:
GET /logs → לקבל את כל הלוגים.

POST /logs → להוסיף לוג.

סיכום:

➕ הוספת אירוע חדש.

📝 עריכת אירוע + צילום + מיקום.

📄 צפיה בלוגים.

✅ פיצ'ר התראה.

✅ פיצ'ר מיקום.
